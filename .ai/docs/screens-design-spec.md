# Code-Word — Описание экранов для дизайна

**Дата:** 2026-06-28  
**Платформа:** Android (Jetpack Compose + Material 3)  
**Язык UI:** Русский  
**Цвета команд:** Красная / Синяя

---

## Навигационный флоу

```
Splash (MainActivity)
    ↓ (анон-авторизация Firebase)
[HOME] Главный экран
    ├─ Создать комнату → [LOBBY] Лобби
    └─ Ввести код → [LOBBY] Лобби
              ↓ (хост нажимает "Начать")
         [GAME] Игровой экран
              ↓ (победитель определён)
         [RESULT] Экран результата
              ├─ "Играть ещё раз" → [LOBBY] (тот же код)
              └─ "Главное меню" → [HOME]
```

---

## Экран 1 — HOME: Главный экран

**Назначение:** Создание новой комнаты или подключение к существующей по коду.

### Структура экрана (сверху вниз)

| # | Элемент | Тип | Описание |
|---|---------|-----|----------|
| 1 | Логотип / заголовок | Text | «Code-Word», крупный, extra bold |
| 2 | Поле «Ваше имя» | OutlinedTextField | Обязательное поле, следующая клавиатура → следующее поле |
| 3 | Кнопка «Создать комнату» | Button (Primary) | Полная ширина |
| 4 | Разделитель «или» | Row | Горизонтальные линии + текст по центру |
| 5 | Поле «Код комнаты» | OutlinedTextField | 6 символов макс, автозаглавные, клавиша «Войти» (IME.Go) |
| 6 | Кнопка «Войти» | Button (Primary) | Полная ширина |
| 7 | Snackbar | SnackbarHost | Отображение ошибок (снизу) |

### Состояния

| Состояние | Что отображается |
|-----------|-----------------|
| **Idle** | Поля активны, кнопки обычные |
| **Loading** | Поля недоступны, в кнопке спиннер вместо текста |
| **Error** | Snackbar с текстом ошибки, возврат в Idle |

### Логика кнопок

- **«Создать комнату»**: активна всегда (имя проверяется при нажатии)
- **«Войти»**: активна только если имя заполнено И код = 6 символов

### Ошибки (Snackbar)

| Ситуация | Текст |
|----------|-------|
| Комната не найдена | «Комната 'XXXXXX' не найдена» |
| Игра уже идёт | «Комната закрыта — игра уже идёт» |
| Игра завершена | «Игра в этой комнате уже завершена» |

## Экран 2 — LOBBY: Лобби

**Назначение:** Выбор роли и команды, отображение готовности игроков, старт игры (хост).

### Структура экрана (сверху вниз)

| # | Элемент | Тип | Описание |
|---|---------|-----|----------|
| 1 | Заголовок «Комната» | Text (small) | Метка |
| 2 | Код комнаты | Text (displayMedium, bold) | Например «ABC123», разреженный трекинг |
| 3 | Подсказка | Text (body) | «Поделитесь кодом с друзьями» |
| 4 | Заголовок «Команды» | Section label | |
| 5 | Блок команд | Row (две колонки) | Красная команда / Синяя команда |
| 6 | Блок «Зрители» | Column (условный) | Видим только если зрители есть |
| 7 | Кнопка «Готов / Не готов» | OutlinedButton | Полная ширина, тогглит состояние |
| 8 | Переключатель приватности | Switch (только хост) | «Закрыть комнату во время игры» |
| 9 | Кнопка «Начать игру» | Button Primary (только хост) | Или «Нужно хотя бы 2 игрока» (disabled) |
| 10 | Сообщение ожидания | Text (не-хост) | «Ожидаем хоста…» |

### Слот-карточка (SlotCard) — × 4

Отображается для каждой роли в каждой команде: Спаймастер Красных, Оперативник Красных, Спаймастер Синих, Оперативник Синих.

| Элемент | Описание |
|---------|----------|
| Метка роли | «Спаймастер» / «Оперативник» |
| Имя игрока | «{имя} ✓» (готов) / «{имя}» (не готов) / «Свободно» (пусто) |
| Фон карточки | Светлый цвет команды — если мой слот; серый variant — если занят другим; обычный surface — если свободен |
| Обводка | 2dp командный цвет — мой слот; 1dp серая — чужой/свободный |
| Кликабельность | Заблокирован, если слот занят другим игроком |

### Состояния экрана

| Состояние | Что отображается |
|-----------|-----------------|
| **Loading** | Полноэкранный CircularProgressIndicator |
| **Ready** | Полный UI, интерактивный |

### Условная видимость

| Элемент | Условие показа |
|---------|---------------|
| Блок «Зрители» | Есть хотя бы один зритель |
| Switch приватности | Текущий пользователь — хост |
| Кнопка «Начать игру» | Текущий пользователь — хост |
| «Ожидаем хоста…» | Текущий пользователь — не хост |
| Кнопка «Начать» (disabled) | Занято < 2 слотов не-зрителями |

### Логика кнопки «Готов»

- Нажатие переключает: Ready → Not Ready → Ready
- Имя игрока в слоте показывает «✓» и меняет цвет (primary/зелёный) при Ready

### Логика кнопки «Начать игру»

- Доступна только хосту
- Активна если ≥ 2 игроков занимают слоты (не-зрители)
- По нажатию: генерирует доску 5×5, выбирает случайную начинающую команду, меняет статус комнаты → PLAYING

## Экран 3 — GAME: Игровой экран

**Назначение:** Основной игровой экран. Отображение доски, управление ходом, ввод подсказок.

### Структура экрана

```
┌─────────────────────────────────┐
│         StatusBar (top)         │
├─────────────────────────────────┤
│                                 │
│       BoardGrid 5×5             │
│     (25 карточек слов)          │
│                                 │
├─────────────────────────────────┤
│  ClueInputPanel (только спаймастер)  │
└─────────────────────────────────┘
```

---

### Компонент: StatusBar

| Элемент | Описание |
|---------|----------|
| Индикатор команды | Цветная точка + «Красные • Подсказка» или «Синие • Угадывание» |
| Статус для игрока | «Ваш ход (подсказка)» / «Ожидаем подсказку...» / «Ваш ход (угадывай)» / «Ожидаем ход команды» |
| Счёт | «● 9» (осталось красных карт) и «● 8» (осталось синих), крупно, полужирный |
| Подсказка (условно) | «{СЛОВО} × {число}» + токены-кружки |

**Токены-кружки (clue tokens):**
- Если count = 0: показать «∞»
- Иначе: до 5 кружков; заполненные — потрачены, полупрозрачные — доступны

---

### Компонент: BoardGrid

Сетка 5×5 = 25 карточек. Отступ между карточками 4dp.

---

### Компонент: CardView (одна карточка)

**Два состояния карточки:**

#### Лицевая сторона — не открыта (Front)

| Роль | Внешний вид |
|------|-------------|
| Спаймастер | Светлый оттенок истинного цвета карты (подсказка) |
| Оперативник / Зритель | Нейтральный серый, слово видно |

- Текст: слово, 11sp, semi-bold
- Кликабельна только для активного оперативника (его ход + фаза Guess + карта не открыта)

#### Обратная сторона — открыта (Back)

| Тип карты | Фон | Текст |
|-----------|-----|-------|
| Красная | Насыщенный красный | Белый |
| Синяя | Насыщенный синий | Белый |
| Нейтральная | Серый | Серый |
| Убийца (Assassin) | Чёрный | Белый |

**Анимация переворота:** 3D flip (0° → 180°, 400ms, FastOutSlowIn)

---

### Компонент: ClueInputPanel

**Показывается только Спаймастеру.**

| Элемент | Тип | Описание |
|---------|-----|----------|
| Заголовок | Text | «Дайте подсказку» |
| Поле «Слово» | OutlinedTextField | Одна строка, Enter = Done |
| Ошибка слова | Supporting text | «Введите слово» (если пустое при отправке) |
| Выбор числа | Row chips | От 1 до N (N = оставшиеся карты своей команды) |
| ChipNumber | Box | Заполненный = выбранный; обводка = не выбранный; цвет команды |
| Кнопка «Отправить» | Button | Активна только когда ход спаймастера (фаза Clue, его команда) |

---

### Оверлей-блокировка

Полупрозрачный белый слой поверх доски — блокирует взаимодействие:
- **Оперативник:** если сейчас не его ход или не фаза Guess
- **Спаймастер:** если сейчас не его команда ходит

---

### Состояния игрового экрана

| Состояние | Описание |
|-----------|----------|
| **Loading** | Полноэкранный спиннер |
| **Clue phase** | Спаймастер вводит подсказку; оперативники ждут |
| **Guess phase** | Оперативники нажимают карточки; спаймастер наблюдает |
| **End** | Победитель определён, переход на RESULT |

---

### Логика хода (упрощённо)

1. **Фаза Clue:** Спаймастер вводит слово + число → отправляет → переход в Guess
2. **Фаза Guess:** Оперативник нажимает карточку:
   - Карта своей команды → открывается, счёт -1; если это последняя — победа (ALL_CARDS)
   - Карта чужой команды или нейтральная → ход переходит другой команде
   - Убийца (Assassin) → противоположная команда побеждает (ASSASSIN)
3. Если исчерпан лимит угадываний → ход переходит другой команде

## Экран 4 — RESULT: Экран результата

**Назначение:** Показать итог игры и предложить следующие действия.

### Структура экрана

| # | Элемент | Тип | Описание |
|---|---------|-----|----------|
| 1 | Фон | Box | Цвет победившей команды, 40% прозрачность |
| 2 | Иконка победителя | Box (80dp круг) | Сплошной цвет команды |
| 3 | Заголовок победы | Text (headlineLarge, extra bold) | «Команда Красных выиграла!» / «Команда Синих выиграла!» |
| 4 | Причина победы | Text (body) | «Открыли все карточки» / «Совершила убийство» |
| 5 | Кнопка «Играть ещё раз» | Button (Primary) | Полная ширина, цвет команды |
| 6 | Кнопка «Главное меню» | OutlinedButton | Полная ширина, цвет команды |

### Логика цветов

Все цвета экрана (фон, кнопки, обводки) определяются командой победителя:
- Победили **Красные** → красная палитра
- Победили **Синие** → синяя палитра

### Причины победы

| Параметр `winReason` | Текст |
|----------------------|-------|
| `ALL_CARDS` | «Открыли все карточки» |
| `ASSASSIN` | «Совершила убийство» |

### Кнопки навигации

| Кнопка | Действие |
|--------|----------|
| «Играть ещё раз» | Переход в Лобби с тем же кодом комнаты |
| «Главное меню» | Переход на главный экран |

## Экран 5 — START: Выбор роли *(не интегрирован)*

**Статус:** Экран реализован, но не включён в навигационный граф. Готов к дальнейшей интеграции.

**Назначение:** Выбор команды (Красные / Синие) и роли (Спаймастер / Оперативник) перед лобби.

### Структура экрана

| # | Элемент | Тип | Описание |
|---|---------|-----|----------|
| 1 | Логотип | Text (displayMedium) | «Code-Word» |
| 2 | Инструкция | Text | «Выберите, за кого играть» |
| 3 | Блок Красных | Column | Два варианта роли |
| 4 | Блок Синих | Column | Два варианта роли |

**Кнопки ролей (по 2 на команду):**

| Кнопка | Тип | Цвет |
|--------|-----|------|
| «Спаймастер» | Button (Filled) | Цвет команды |
| «Оперативник» | OutlinedButton | Цвет команды |

При нажатии — немедленный callback `onRoleSelected(role, team)`, без загрузки.

## Цветовая система

### Карточки

| Тип | Тёмный (revealed) | Светлый (hint, unrevealed for spymaster) |
|-----|-------------------|------------------------------------------|
| Красная | `CardRed` | `CardRedLight` |
| Синяя | `CardBlue` | `CardBlueLight` |
| Нейтральная | `CardNeutral` (серый) | `CardNeutralLight` |
| Убийца | `CardAssassin` (чёрный) | `CardAssassinLight` (тёмно-серый) |
| Не открыта (оперативник) | — | `CardUnrevealed` (серый) |

### Команды

| Команда | Primary | Light |
|---------|---------|-------|
| Красная | Насыщенный красный | Бледно-розовый |
| Синяя | Насыщенный синий | Бледно-голубой |

## Роли и их UX-ограничения

| Роль | Видит цвета карт | Нажимает карты | Вводит подсказку | Наблюдает |
|------|-----------------|----------------|------------------|-----------|
| **Спаймастер** | Да (светлые тона) | Нет | Да | Своя фаза |
| **Оперативник** | Нет (серые) | Да (свой ход) | Нет | Ход врага |
| **Зритель** | Нет (серые) | Нет | Нет | Всегда |
